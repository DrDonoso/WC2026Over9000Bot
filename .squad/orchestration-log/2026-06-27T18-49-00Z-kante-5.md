## Implementation Summary

### Part 1 — Seed at 0-0 at Kickoff
**File:** `src/worldcup_bot/__main__.py:poll_kickoff_job`
- Seeds `live_scores[mid] = {home:0, away:0, status:IN_PLAY}` in finally block after announcing kickoff
- Idempotent (only if key absent)
- Immediate `save_scores` call

### Part 2 — Catch-up Recovery from Reddit Thread
**File:** `src/worldcup_bot/__main__.py:_attempt_goal_recovery`
- New async function called by `_process_goal_delta` when `delta.kind == "catchup"`
- Tries cached thread lookup, falls back to HTTP search (5s timeout)
- Parses `GoalEvent` objects from thread
- For each missed goal: find matching event by score and team
- If ALL matched: emit per-goal `_notify_goal` (proper notifications)
- If ANY failed: fall back to neutral `_notify_catchup`
- **Rule:** ALL-proper or ALL-neutral, never mixed
- Set `seen_thread[match_key]` for dedup after recovery

### Part 3 — FINISHED Two-Tick Eviction
**File:** `src/worldcup_bot/__main__.py:poll_goals_job`
- Track `was_already_finished = (stored.get("status") == "FINISHED")`
- In `else:` branch (no-delta): if `was_already_finished` → evict from `live_scores`, `seen_api`, `seen_thread`
- First FINISHED tick (IN_PLAY→FINISHED): no eviction
- Second FINISHED tick (FINISHED→FINISHED, no delta): evict
- FT goals (first FINISHED with delta) go to `elif deltas:` → announce then save status

### Part 4 — Immediate Save in poll_thread_goals_job
**File:** `src/worldcup_bot/__main__.py:poll_thread_goals_job`
- Moved `save_scores(state_path, scores)` inside `goal_lock`, immediately after score claim
- Removed deferred save at end of loop
- Effect: score persistence atomic with in-memory claim

### Part 5 — 5s Timeout on find_match_thread
**File:** `src/worldcup_bot/reddit/scanner.py`
- Reduced HTTP timeout from 15s to 5s
- Prevents `_attempt_goal_recovery` from hanging poll_goals_job
