## Summary

Technical review & design gate for Kanté's goal-notification bug fixes. Four bugs identified, three immediate fixes approved + one refinement required.

### Review Decisions

#### Decision 1: Catch-Up Message Misinformation
- **Problem:** Old code fabricated intermediate scorelines + per-goal attribution when bot missed goals.
- **Approved approach:** ONE neutral catch-up message per match (no fabrication, final score only).
- **Spec:** `⚠️ Me perdí N gol(es) / 🇪🇨 Ecuador 1-1 Germany 🇩🇪`
- **Rationale:** Honest admission of delay > confident wrong score. Users trust honest bots more.

#### Decision 2: Race Condition Fix (Reordering + Hardening)
- **Problem:** `poll_goal_clips_job` set status AFTER edit; `_backfill_scorer_in_clip_store` gap could clear keyboard.
- **Approved:** Reorder status-set BEFORE edit (correct per asyncio single-thread guarantee).
- **Hardening required:** Omit `reply_markup` when status ≠ "ready" (NOT pass `None`, which removes keyboard).
- **Status:** Both implemented by Kanté in refinement round.

#### Decision 3: Delete-After-Send
- **Problem:** None identified. Ordering is correct (file_id → persist → unlink).
- **Approved:** Ship as-is. Safety properties verified (no crash scenarios).

#### Decision 4: Multi-Goal Restart Collision (Design Limitation)
- **Issue:** Reconcile restart emits N deltas all with `new_home/new_away` final score → token key collision on same-team multi-goals.
- **Effect:** 3 notifications sent correctly, but clip-store has 2 entries (first overwritten). First message's button links to wrong clip.
- **Recommendation:** Emit incremental scores (like `__main__.py` does) to avoid collision.
- **Remark:** Rare scenario (bot restart + missed 2+ home goals); no crash, just suboptimal clip mapping.
