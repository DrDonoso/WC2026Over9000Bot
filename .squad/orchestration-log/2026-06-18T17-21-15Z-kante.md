## kante-43: /endirecto 429 Fix

**Problem:** /endirecto showed NO inline keyboard in production.

**Root Cause:**
- `cmd_en_directo` created fresh RedditMatchScanner each call (no cache)
- `find_match_thread` calls `/search` endpoint â†’ Reddit 429 Too Many Requests
- Handler fell back to plain format (no keyboard)

**Solution:**

1. **TTL In-Memory Cache** (`src/worldcup_bot/reddit/scanner.py`):
   - `get_match_threads()`: 30s cache; on 429 returns stale or `[]`
   - `get_thread_body(permalink)`: 90s cache per-permalink; on 429 returns stale or `""`
   - **Never raises** â€” graceful degradation

2. **New Method** `find_thread_permalink(home_name, away_name) â†’ str | None`:
   - Uses reliable `/new/` listing (not `/search`)
   - Returns cached thread's permalink or `None`

3. **Updated Handler** (`src/worldcup_bot/bot/handlers.py`):
   - `cmd_en_directo` reuses `context.bot_data["reddit_scanner"]`
   - Tries `find_thread_permalink` (cached /new/) first
   - Falls back to `find_match_thread` (search) if needed

**Net Effect:** Goal poller's recent fetches are reused by /endirecto â†’ no new Reddit hit â†’ no 429 â†’ keyboard shown.
