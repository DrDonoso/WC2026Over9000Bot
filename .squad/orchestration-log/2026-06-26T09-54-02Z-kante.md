## Summary

Investigated and fixed 4 live bugs in the goal-notification pipeline during Ecuador–Germany and Tunisia–Netherlands matches:

### Bug A1 (Missed goals) — API status-flip delay + restart mid-match
- Root cause: `poll_goals_job` guards on `IN_PLAY`/`PAUSED`; API delay means non-zero scores are not announced on initial match-status flip.
- Restart case: reconcile() had a blind seed pass that lost intermediate goals.
- Fix: Emit ONE neutral `GoalDelta(kind="catchup")` instead of N fabricated per-team goals.

### Bug B1 (Missing keyboards) — Race condition
- Root cause: `poll_goal_clips_job` set status AFTER `edit_message_reply_markup`.
- Gap window: `_backfill_scorer_in_clip_store` could read stale status, call `edit_message_text(reply_markup=None)`, clearing keyboard.
- Fix: Set status BEFORE edit. Harden backfill to omit `reply_markup` key when status ≠ "ready" (preserve existing).

### Bug B2 (Disk-full failures) — Mitigated by delete-after-send
- Root cause: Clips accumulate; prune is 7 days; ~4GB volume fills.
- Fix: Delete local file AFTER successful send + file_id persisted (new feature).

### Bug D (New feature) — Delete clip after successful send
- Safety: file_id cached first, then file deleted with `missing_ok=True`.
- Graceful: stale file_id on retry → falls back to error message (no crash).
