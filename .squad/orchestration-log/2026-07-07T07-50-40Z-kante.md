## Context

Incident: USA vs Belgium match (2026-07-06) generated 100+ alternating "⚽ GOOOOL!" and "❌ Gol anulado" messages.

Root cause: Cross-source score reconciliation bug where one source (Reddit thread, fast ~25s) announces a VAR-disallowed goal before the other source (API, slow ~60s) has ever seen the goal. The lagging source's later catch-up to the goal is mistaken for a new goal → announces it → catches up to VAR → announces disallowed → loop repeats every tick.

## Implementation

### Fix in `src/worldcup_bot/__main__.py`

**In `poll_thread_goals_job` (thread-sourced disallowed):**
- After a disallowed delta, advance API's seen baseline: `seen_api[key] = {max(cur["home"], ann_homeaway["home"]), max(cur["away"], ann_homeaway["away"])}`
- This prevents the API's later catch-up to the pre-VAR score from being mistaken for a new goal.

**In `poll_goals_job` (API-sourced disallowed):**
- Same fix in reverse: after an API disallowed, advance thread's seen baseline.

**Atomic execution:** Both fix locations execute inside `goal_lock` synchronously.
